# As regras de negócio do sistema

O sistema deve impedir:

1. Que seja registrada uma ocorrência sem descrição, bairro ou origem;
2. Criar ocorrência sem prioridade existente;
3. Aceitar ocorrência sem setor responsável;
4. Recusar ocorrência sem justificativa;
5. Marcar ocorrência como duplicada sem protocolo relacionado;
6. Executar ocorrência onde ainda existe vistoria pendente;
7. Encerrar ocorrência sem registro de atendimento, vistoria ou justificativa adequada;
8. Alterar uma ocorrência encerrada;
9. Registrar movimentação com data anterior à abertura;
10. Registrar movimentação incompatível com a situação atual;
